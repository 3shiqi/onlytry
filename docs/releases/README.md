# Releases

Each file in this directory describes one formal project version.

Naming rule:

- `0.1.0.md`
- `0.1.1.md`
- `0.2.0.md`
- `0.2.1.md`
- `0.3.0.md`
- `0.4.0.md`
- `0.5.0.md`
- `0.6.0.md`

Use release files for:

- what shipped in that version
- migration notes
- rollout notes
- notable risks or constraints
