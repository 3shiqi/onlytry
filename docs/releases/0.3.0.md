# Release 0.3.0

Date: 2026-04-24

## Summary

Introduced the first global training-state layer for dual-mode operation and TSS-driven periodization.

## Included

- `PRESCRIPTIONS` taxonomy for granular training categories
- `TRAIN` and `PLAY` app modes
- `currentTSS` and `externalLogs` global state
- `calculatePlayTSS()` and `calculateFluidCalendar()` pure helpers
- root provider wiring through `src/main.jsx`
- workout engine support for prescription-based quota resolution

## Behavioral Notes

- the 7-day fluid calendar is forward-looking and starts with tomorrow
- if TSS exceeds `80`, tomorrow is forced to `无痛重启 (Recovery)`
- if TSS exceeds `50`, tomorrow is forced to `足踝稳定/肩胸功能`
